Activity Recognition Dashboard
MHEALTH Dataset · 1.2M Records · 12 Activities · 3 Sensor Devices
"The body never lies —
machines can now read it."
— Inspired by Martha Graham · Built with ML + Spring Boot + Python
HAR System is a full-stack Human Activity Recognition platform trained on the MHEALTH dataset (1.2M records, 12 activities, 10 subjects). It uses 5 ML models — Logistic Regression, KNN, Decision Tree, Random Forest, and a 1D CNN — to classify body movements from wearable sensor data in real time. Built with Python Flask (ML inference), Spring Boot (REST API + MySQL persistence), and a vanilla JS frontend.
Python · Flask Spring Boot · MySQL TensorFlow · 1D CNN MHEALTH Dataset scikit-learn
▼ HOW TO USE THIS DASHBOARD
· 2 min read · For interviewers & new users
01
🎯 Run a Prediction
Go to Predict or use the Sensor Input panel on this dashboard. Enter 12 sensor values (ALX, ALY, ALZ from chest accelerometer; GLX–GLZ from chest gyroscope; ARX–ARZ from wrist; GRX–GRZ from ankle). Hit Load Sample Data to auto-fill realistic values, then click Run Prediction.
02
🤖 Choose Your Model
Select from 5 trained classifiers: Logistic Regression (64.87%), KNN (97.54%), Decision Tree (93.42%), Random Forest (97.96%), or 1D CNN (97.93% F1). The CNN uses TensorFlow and gives the best F1-Macro score across all 12 activities.
03
📊 View Analytics
The Analytics page stores your full prediction history locally — it survives page refresh. Filter by today, this week, or activity type. Export your session data as CSV or JSON for further analysis.
04
🔬 Explore Models
The Models page has an interactive 12×12 Confusion Matrix — hover any cell to see actual vs predicted counts. Switch between RF, CNN, and KNN matrices using the dropdown. The radar chart compares models across accuracy, speed, interpretability, and scalability.
05
📡 Live Sensor Feed
Go to Sensors and click ▶ Simulate Live to watch all 12 sensor channels animate in real time. The waveform shows chest accelerometer (ALX/Y/Z) and gyroscope (GLX) signals with realistic motion patterns.
06
💾 Dataset Info
Visit MHEALTH to explore the dataset details — sensor placement, selected features, preprocessing pipeline, and the class distribution of all 12 activities across 10 subjects recorded at 50Hz sampling rate.

Dataset Overview
The MHEALTH (Mobile Health) dataset comprises body motion and vital signs recordings of 10 volunteers performing 12 physical activities. Sensors were placed on the chest, right wrist, and left ankle capturing tri-axial accelerometer and gyroscope data at 50Hz.
Sensor Placement
Chest: Accelerometer (ALX, ALY, ALZ) + Gyroscope (GLX, GLY, GLZ)

Right Wrist: Accelerometer (ARX, ARY, ARZ)

Left Ankle: Gyroscope (GRX, GRY, GRZ)
Selected Features (12)
ALXALYALZ GLXGLYGLZ ARXARYARZ GRXGRYGRZ
Preprocessing Steps
Missing value removal · StandardScaler normalization · Label encoding · Train/test split (80/20) · Stratified sampling to preserve class balance
